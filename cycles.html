Task № 14

Укажите какие значения будут выведены в консоль, в случаях использования инкремента в префикской и постфиксной формах. Будет ли результат отличаться?

							
// Префиксная форма
let i = 0
while (++i < 5) {
  console.log(i)
}
// Постфиксная форма
let i = 0
while (i++ < 5) {
  console.log(i)
}
							
						
							
// Префиксная форма инкремента:
let i = 0
while (i++ < 5) {
  console.log(i)
}
// Результат: от 1 до 4.
// Первое значение: i = 1, так как операция ++i сначала увеличит i, а потом уже произойдет сравнение и выполнение alert.
// Далее 2, 3, 4... значение выводятся одно за другим. Для каждого значения сначала происходит увеличение, а потом - сравнение, так как ++ стоит перед переменной.
// При i = 4 произойдет увеличение i до 5, а потом сравнение while (5 < 5) -> false. Поэтому на этом цикл остановится, и значение 5 не будет выведено в модальное окно.


// Постфиксная форма инкремента:
let i = 0
while (++i < 5) {
  console.log(i)
}
// Результат: от 1 до 4.
// Первое значение i = 1. Оператор i++ увеличивает i, возвращая старое значение, так что в сравнении с i++ < 5 будет участвовать старое i = 0.
// На последующий вызов alert уже не относится к этому выражению, так что получит новый i = 1.
// Далее 2, 3, 4... Для каждого значения сначала происходит сравнение, а потом - увеличение, и затем выводит значение в консоль.
// Окончание цикла: при i = 4 произойдет сравнение while (4 < 5) - верно, после этого сработает i++, увеличив i до 5, так что значение 5 будет выведено.
// Значение i = 5 последнее, потому что на следующем шаге while (5 < 5) -> false.

Task № 15

Укажите какие значения будут выведены в консоль, в случаях использования инкремента в префикской и постфиксной формах. Будет ли результат отличаться?

							
// Постфиксный вариант i++
for (let i = 0; i < 5; i++) {
  console.log(i)
}
// Префиксный вариант ++i
for (let i = 0; i < 5; ++i) {
  console.log(i)
}

Ответ: от 0 до 4 в обоих случаях.

								
// Постфиксный вариант i++
for (let i = 0; i < 5; i++) {
  console.log(i)
}
// Префиксный вариант ++i
for (let i = 0; i < 5; ++i) {
  console.log(i)
} 

// Такой результат обусловлен работой алгоритма for:
// 1. Выполнить присваивание i = 0, перед чем-либо;
// 2. Проверить условие i < 5;
// 3. Если true - выполнить код в теле цикла, а затем i++;
// 4. Увеличиние i++ выполняется отдельно от проверки условия (2), значение i при этом не используется. 
// Поэтому нет никакой разницы между префиксной и постфиксной формой оператор инкремент.

Task № 19

Натуральное число, больше 1, называется простым, если оно ни на что не делится, кроме себя и 1.

Другими словами, n > 1 - простое, если при его делении на любое число кроме 1 и n есть остаток. Например 5 это простое число, оно не может быть разделено без остатка на 2, 3 и 4.

Напишите код, который выводит все простые числа из интервала от 2 до n.

Для n = 10 результат должен быть 2, 3, 5, 7.

P.S. Код ниже должен легко модифицироваться для любых других интервалов.

								
// Существует множество алгоритмов решения этой задачи. В данном случае воспользуемся вложенными циклами.
// Для всех i от 1 до 10 {
//   проверить, делится ли число  i на какое-либо из чисел до него
//   если делится, то это i не подходит, берем следующее
//   если не делится, то i - простое число
// }

// Решение с использованием метки:
let n = 10

nextPrime:
for (let i = 2; i <= n; i++) { // Для всех i от 1 до 10
  for (let j = 2; j < i; j++) { // проверить, делится ли число i на какое-либо из чисел до него. Если делится, то это i не подходит, берем следующее, 
				//если не делится, то i - простое число
    if (i % j == 0) continue nextPrime // не подходит, берём следующее
  }
  console.log(i) // простое число
}

// Конечно же, его можно оптимизировать с точки зрения производительности. Например, проверять все j не 2 до i, а от 2 до квадратного корня из i. 
//А для больших чисел - существуют более эффективные специализированные алгоритмы проверки простоты числа, к примеру: квадратное решето и решето числового поля.

  case 0 :
    console.log('Вы ввели число 0')
    break

  case 1 :
    console.log('Вы ввели число 1')
    break

  case 2 :
  case 3 :
    console.log('Вы ввели число 2, а может 3')
    default :
    console.log('Введено не правильное значение')
    break
}
