Математические термины

  • Натуральное число - это число, которое используется для подсчета чего-то конкретного, осязаемого (от 1 до 999+).
  • Целое число - это натуральное число, а также противоположное ему число и 0 (-7, 222, 0), не имеющее дробной части.
  • Простое число - натуральное (целое положительное) число, имеющее ровно два различных натуральных делителя — 1 и самого себя. Другими словами, число является простым, если оно больше 1 и при этом делится без остатка только на 1 и на само себя.

Fundamental concepts:

  • Информатика (computer science) - наука о методах и процессах сбора, хранения, обработки, передачи, анализа и оценки информации с применением компьютерных технологий, обеспечивающих возможность ее использования для принятия решений.
  • Модальный метод - останавливает выполнение скриптов и не позволяют пользователю взаимодействовать с остальной частью веб-страницы до тех пор, пока окно не будет закрыто.
  • Вычислительная машина - механизм, электромеханическое или электронное устройство, предназначенное для автоматического выполнения математических операций.
  • Машинный код - система команд (набор кодов операций) конкретной вычислительной машины, которая интерпретируется непосредственно процессором или микропрограммами этой вычислительной машины.
  • Синтаксис языка программирования - это набор правил, описывающий комбинации символов алфавита, считающиеся правильно структурированной программой (документов) или ее фрагментом.
  • Семантика (semantics) - дисциплина, изучающая формализацию значений конструкций языков программирования посредством построения их формальных математических модулей. Другими словами, семантика - это  начальное смысловое значение statements, основных конструкций языка и т.п.
  • Mодель, представление, контролер (MVC (Model View Controler)) - cхема разделения данных приложения, и управляющей логики на три отдельных компонента: модель, представление и контроллер — таким образом, что модификация каждого компонента может осуществляться независимо. Модель предоставляет данные и реагирует на команды контроллера, изменяя своё состояние.
  • Низкоуровневый язык программирования — язык программирования, близкий к программированию непосредственно в машинных кодах используемого реального или виртуального (например, байт-код, IL) процессора.
  • Теория языков программирования - раздел информатики, посвященный вопросам проектирования, анализа, определения характеристик и классификации языков программирования и изучением их индивидуальных особенностей.
  • Структура данных - программная единица, позволяющая хранить и обрабатывать множество однотипных и(или) логически связанных данных в вычислительной машине. Для добавления, поиска, изменения и удаления данных, структура данных предоставляет некоторый набор функций, составляющий ее интерфейс.
  • Очередь (queue) - абстрактный тип данных с дисциплиной доступа к элементам "первый пришел - первый вышел". Добавление элемента (принято обозначать словом "enqueue" - поставить в очередь) возможно лишь в конец очереди, выборка - только из начала очереди (что принято называть словом "dequeue" - убрать из очереди), при этом выбранный элемент из очереди удаляется.
  • Стек (double ended queue) - абстрактный тип данных, в котором элементы можно добавлять и удалять как в начало так и в конец. Может быть реализована при помощи двусвязного списка (структура данных).
  • Двусторонняя очередь (stack) - абстрактный тип данных, представляющий собой список элементов организованных по принципу LIFO (от англ - last in - first out, "последний пришел - первым вышел"). Чаще всего принцип работы стека сравнивают со стопкой тарелок: чтобы взять вторую сверху, нужно снять верхнюю.
  • Файл - специально организованная структура данных, распознаваемая компьютером как единое целое.
  • Типизация данных - возможность, в определенном языке программирования, переводить один тип данных в другой.
  • Статическая типизация — приём, широко используемый в языках программирования, при котором переменная, параметр подпрограммы, возвращаемое значение функции связывается с типом данных в момент объявления и тип не может быть изменён позже.
  • Слабая типизация - по одной из классификаций, языки программирования неформально делятся на сильно и слабо типизированные, то есть обладающие сильной и слабой системой типов. Эти термины не являются однозначно трактуемыми, и чаще всего используются для указания на достоинства и недостатки конкретного языка. Существует более конкретные понятия, которые и приводят к называнию тех или иных систем типов сильными или слабыми.
  • Динамическая типизация - прием, используемый в языках программирования и языках спецификации, при котором переменная связывается с типом данных в момент присваивания значения, а не в момент объявления переменной. Таким образом, в различных участках программы одна и та же переменная может принимать значения разных типов данных. Динамическая типизация упрощает написание программ для работы с меняющимся окружением, при работе с данными переменных; при этом отсутствие информации о типе на этапе компиляции повышает вероятность ошибок в используемых модулях.
  • Динамический язык программирования — язык программирования, который позволяет определять типы данных и осуществлять синтаксический анализ и компиляцию «на лету», на этапе выполнения программы. Динамические языки удобны для быстрой разработки приложений.
  • Декларативное программирование — парадигма программирования, в которой задаётся спецификация решения задачи, то есть описывается, что представляет собой проблема и ожидаемый результат.
  • Парадигма программирования — это совокупность идей и понятий, определяющих стиль написания компьютерных программ (подход к программированию). Это способ концептуализации, определяющий организацию вычислений и структурирование работы, выполняемой компьютером.
  • Автоматическое управление памятью (сборка мусора) - специальный процесс называемый сборщиком мусора, периодически освобождает память, удаляя объекты, которые уже не будут востребованы приложениями.
  • Объектно-ориентированное программирование - это подход, при котором вся программа рассматривается как набор взаимодействующих друг с другом объектов. При этом важно знать их характеристики. У каждого объекта в системе есть свойства и поведение, как и у любого реального объекта.
  • Прототипное программирование - стиль объектно-ориентировочного программирования, при котором отсутствует понятие класса, а наследование производится путем клонирования существующего экземпляра объекта – прототипа.
  • Наследование (inheritance ) - это процесс, посредством которого один объект может приобретать свойства другого. Точнее, объект может наследовать основные свойства другого объекта и добавлять к ним черты, характерные только для него.
  • Прототип (prototype) - это объект, который в основном представляет собой контейнер для хранения свойств и методов, которые требуется наследовать объектам, расположенными дальше по цепочке прототипов.
  • Арифметическая операция (arithmetic operation) - операция с цифрами (сложение, вычитание, умножение и деление).
  • Цифры - система знаков для записи конкретных значений чисел. Означает один из 10 знаков (от 0 до 9).
  • Число - одно из основных понятий математики, используемое для количественной характеристики, сравнения, нумерации объектов и их частей.
  • Натуральный ряд - последовательность всех натуральных чисел, расположенных в порядке возрастания.
  • Вещественное (действительное) число - математический объект, возникший из потребности измерения геометрических и физических величин, а также проведения таких вычислительных операций, как извлечение корня, вычисление логарифмов, решение алгебраических уравнений, исследование поведения функций.
  • Рациональное число - это число, которое можно представить в виде положительной или отрицательной обыкновенной дроби или числа ноль. Если число можно получить делением двух целых чисел, то это число рациональное.
  • Иррациональное число - это вещественное число, которое не является рациональным, то есть не может быть представлено в виде обыкновенной дроби, но может быть представлено в виде бесконечной непериодической десятичной дроби.
  • Простая (обычная) дробь - это запись вида x\y, где x & y любые натуральные числа.
  • Десятичная дробь - это число с остатком, где остаток стоит после целой части и разделяется запятой.
  • Смешанная дробь - это число с остатком, но остаток записывается в виде простой дроби (с черточкой).
  • Бесконечная непериодическая десятичная дробь - это бесконечная дробь, у которой одна цифра или группа цифр повторяются. Повторяющаяся группа цифр называется периодом и записывается в скобках. Число, которое можно записать в виде бесконечной непериодической десятичной дроби, называется иррациональным числом.
  • Аппроксимация (приближение) - научный метод, заключающийся в замене одних объектов другими, в каком-то смысле близкими к исходным, но более простыми. Аппроксимация позволяет исследовать числовые характеристики и качественные свойства объекта, сводя задачу к изучению более простых или более удобных объектов.
  • Модуль - функционально законченный фрагмент программы, оформленный в виде отдельного файла с исходным кодом или поименованной непрерывной ее части.
  • Объектный модуль - файл с промежуточным представлением отдельного модуля программы, полученный в результате обработки исходного кода компилятором.
  • Модуль числа – это расстояние от нуля до данного числа. Например, |-5| = 5. То есть расстояние от точки -5 до нуля равно 5.
  • Тип данных - фундаментальное понятие теории программирования. Тип данных определяет множество значений, набор операций, которые можно применять к таким значениям и, возможно, способ реализации хранения значений и выполнения операций.
  • Данные - это форма представления информации (сведения), необходимая для получения какого-то вывода, или решения определенной задачи.
  • JavaScript - это мультипарадигменный язык программирования. Поддерживает Объектно-ориентированный, императивный и функциональный стили. Имеет динамическую типизацию, и автоматическое управление памятью (очистка).
  • Скрипт - представляет собой программный код - набор statements, который не требует предварительной обработки (например, компиляции) перед запуском. Код JS интерпретируется движком браузера во время загрузки веб-страницы.
  • Примитивный тип данных - неизменяемый тип данных.
  • Объектный тип данных - изменяемый тип данных, то есть значение изменяемого типа можно менять. Программа JS способна изменить значение свойств объектов и элементов массивов.
  • Инструкция – это синтаксическая единица языка программирования, которая выражает действие, которое необходимо выполнить. Все программы в JS состоят из инструкций, а сами инструкции могут состоять из значений, операторов, выражений, ключевых слов и комментариев.
  • Оператор (operator) - это символы языка, выполняющие различные операции с данными. Операторы могут записываться с помощью символов пунктуации или ключевых слов.
  • Унарный оператор (unary operator) - оператор, который применяется только к одному операнду.
  • Бинарный оператор (binary operator) - оператор, который применяется к двум операндам.
  • Тернарный оператор (ternary operator) - оператор, который применяется к трем операндам. В языке JS существует один тернарный оператор - ?.
  • Приоритет операторов - определяет порядок, в котором операторы выполняются. Операторы с более высоким приоритетом выполняются первыми.
  • Ассоциативность (associativity) - свойство операций в языках программирования, позволяющее восстановить последовательность их выполнения при отсутствии явных указаний на очередность при равном приоритете. Другими словами - определяет порядок, в котором обрабатываются операторы с одинаковым приоритетом.
  • Операнд (operand) - различные значения: то, к чему применяются операторы, например строки, числа, переменные, константы и прочие объекты, которые могут участвовать в выражении.
  • Выражение (expression) - это совокупность значений, операндов и операторов, которые либо присваивают переменной значение, либо возвращают какое-то значение без его присвоения. Бывают простыми (первичными) и сложными (составными). Результатом вычисления выражения является значение определенного типа данных.
  • Простое выражение (primary expression) - выражение которое не включает в себя другие выражения.
  • Сложное выражение (complex expression) - выражение которое включает в себя другие выражения.
  • Значение (value) - результат вычисления выражения.
  • Ключевое слово - это слово которое существует в ядре языка, встроенные в его синтаксис и имеющее специальное значение.
  • Зарезервированные слова  — это слова, которые пока еще не существуют в ядре языка JS и не встроены в его синтаксис, но в будущем, эти слова могут быть внедрены в ядро JS.
  • Программа – это последовательность инструкций (команд), которые указывают браузеру выполнить какие-то действия.
  • Директива (от англ. directive -команда или указание) - используется для описания некоторых конструкций языка программирования, то есть указаний компилятору особенностей обработки при компиляции.
  • Ассемблер - транслятор программы из текста на языке ассемблера, в программу на машинном языке.
  • Язык ассемблера - машинно-ориентировочный язык программирования низкого уровня. Представляет собой систему обозначений, используемую для представления в удобно читаемой форме программ, записанных в машинном коде.
  • Исходный код - любые исходные данные для транслятора. Исходный код транслируется в исполняемый код целиком до запуска программы при помощи компилятора или может исполняться сразу при помощи интерпретатора (текст компьютерной программы на каком-либо языке программирования или языке разметки, который может быть прочтен человеком).
  • Компиляция – трансляция (преобразование) программы, составленной на исходном языке высокого уровня, в эквивалентную программу на низкоуровневом языке, близком машинному коду, выполняемая компилятором.
  • Компилятор – это программа, которая переводит текст, написанный на языке программирования, в набор машинных кодов.
  • Интерпретация (interpreter) — построчный анализ, обработка и выполнение исходного кода программы или запроса (в отличие от компиляции, где весь текст программы, перед запуском, анализируется и транслируется в машинный или байт-код, без её выполнения).
  • Интерпретатор (interpreter) - программа (разновидность транслятора) выполняющая интерпретацию (в случае с JS - это браузер). Интерпретатор считывает каждую строчку кода, и построчно выполняет данный код. Если допустить ошибку, то программа остановится и перестанет выполняться.
  • Транслятор — программа или техническое средство, выполняющее трансляцию программы.
  • Трансляция программы — преобразование программы, представленной на одном из языков программирования, в программу на другом языке. Транслятор обычно выполняет также диагностику ошибок, формирует словари идентификаторов, выдаёт для печати текст программы и т. д.
  • Транспиляция - преобразование программы, при котором используется исходный код программы, написанной на одном языке программирования в качестве исходных данных, и производится эквивалентный исходный код на другом языке программирования.
  • Транспайлер, транспилер (transpiler) — программа (тип компилятора), выполняющая транспиляцию программы.
  • Лексема (lexeme) - минимальная единица языка, имеющая значение. Последовательность машинных символов исходного кода программы, имеющие определенное совокупное значение.
  • Литерал (literal) - это значение данных, находящееся прямо в программе. Когда парсер видит идентификатор, ему сначала нужно вычислить его значение, а в случае с литералом значение этого выражения будет таким же, каким его видно в ходе программы.
  • Идентификатор (identifier) - это просто имя. Идентификаторы в JS применяются для именования переменных, свойств, функций и классов, а также для того, чтобы снабдить метками некоторые циклы в коде.
  • Unicode - стандарт кодирования символов, включающий в себя знаки почти всех письменных языков мира.
  • Эмотикон (emoticon)  - пиктограмма, изображающая эмоцию; чаще всего составляется из типографических знаков.
  • Объект (Object) - это один из способов хранения данных. Объекты состоят из набора свойств (ключ: значение). Используются для хранения коллекций различных значений, и более сложных вещей.
  • Свойство - это значение, ассоциированное с объектом JS.
  • Метод - это действия, которые можно выполнить с объектами. Метод JS это свойство, содержащее определение функции. Методы это функции, хранящиеся как свойства объекта.< /li>
  • Math - это встроенный объект, который предоставляет математические операции и константы.
  • Двоичные данные (бинарные) - это информация, представленная в виде 0 и 1.
  • Массив (array) – особая структура данных (специальный вид объекта), которая представляет упорядоченную коллекцию пронумерованных значений.
  • Типизированный массив - является массивоподобным объектом, предоставляющий механизм доступа к сырым двоичным данным.
  • Отладка - этап разработки компьютерной программы, на котором обнаруживают, локализуют и устраняют ошибки.
  • Инкремент ++ (increment) - это операция, которая выполняет увеличение переменной на единицу.
  • Декремент -- (decrement) - это операция, которая выполняет уменьшение переменной на единицу.
  • Постфиксная форма оператора (postfix operator form) - когда оператор располагается после переменной. В этом случае, сначала выполняется присваивание (возврат значения), а после чего значение переменной изменяется на 1. Используется в случаях, когда нужно увеличить или уменьшить значение переменной на 1, при этом получить значение до выполнения оператора.
  • Префиксная форма оператора (operator prefix form) - когда оператор располагается до переменной. В этом случае, сначала значение переменной изменяется на 1, а затем происходит присваивание (возврат значения). Используется в случаях, когда нужно сразу же получить результат работы оператора инкремента или декремента.
  • Ключевое слово (keyword) - предназначено, чтобы определить какое действие JS необходимо выполнить, например ключевое слово let - указывает браузеру создать переменную.
  • Метка (label) - имеет вид идентификатора с двоеточием перед циклом. Единственный способ для директив break / continue выйти за пределы текущего цикла, повлиять на выполнение внешнего цикла. Также метки не позволяют "прыгнуть" в произвольное место кода, в JS нет такой возможности.
  • Цикл (cycle) - обеспечивает многократное выполнение повторяющихся вычислений. Служит для оптимизации процесса написания кода, выполняя одну и ту же инструкции, или блок инструкций, образующих тело цикла, заданное число раз (используя переменную - счетчик), или пока заданное условие имеет значение true. Циклы выполняют обход последовательности значений. На производительность цикла влияют количество итераций и количество операций, выполняемых в теле цикла каждой итерации.
  • Итерация (iteration) - однократное выполнение цикла.
  • Конкатенация (concatenation) - операция соединения значений с типом String. Для конкатенации, как и для сложения чисел, используется оператор + (сложение).
  • Функция высшего порядка - функция, принимающая в качестве аргументов другие функции или возвращающая другую функцию в качеств результата.
  • Функции первого класса - это неотъемлемая частью функционального программирования, в котором использование функций высшего порядка является стандартной практикой. В частности, это означает, что язык поддерживает передачу функций в качестве аргументов другим функциям, возврат их как результат других функций, присваивание их переменным или сохранение в структурах данных. В языках с функциями первого класса имена функций не имеют никакого специального статуса, они рассматриваются как обычные значения, тип которых является функциональным.
  • Функция - это ключевая концепция в JS. Специальный тип объектов, позволяющий формализовать средствами языка определенную логику поведения и обработки данных. Блок кода, который можно использовать много раз.
  • Сигнатура функции - формальное описание типов аргументов и типа возвращаемого значения функции.
  • Классы - 
  • Концепция – это систематизация всех идей (взглядов), выработанных для понимания направления развития. Она дает ответ на вопрос — как достичь намеченной цели. По сути, это единый определяющий замысел (стратегия) для решения любой задачи.
  • Концепт - инновационная идея, содержащая в себе созидательный смысл. Продукт, демонстрирующий эту идею, называют концепт-продукт, то есть выпускаемая производителем в единственном экземпляре модель, предназначенная для демонстрации общественности.
  • Концептуализация - это процесс определения набора когнитивных признаков (в том числе — и категориальных) какого-либо явления реального или воображаемого мира, которые позволяют человеку хранить в сознании и пополнять новой информацией сколько-нибудь очерченное понятие и/или представление об этом явлении и отличать его от других феноменов.
  • Когнитивный признак — это отдельный признак объекта, осознанный человеком и отображенный в структуре соответствующего концепта как отдельный элемент его.
  • Стратегия — общий, не детализированный план, охватывающий длительный период времени, способ достижения сложной цели какой-либо деятельности человека. Задачей стратегии является эффективное использование наличных ресурсов для достижения основной цели.
  • Формализация — представление какой-либо содержательной области (рассуждений, доказательств, процедур классификации, поиска информации, научных теорий) в виде формальной системы или исчисления. ... Принято говорить не «искусственный язык логики», а «формализованный язык логики».
  • Интерфейс - это "проводник" между человеком и программой, операционной системой, техническим устройством или способ взаимодействия приложений между собой. Человек дает команды с помощью интерфейса, устройство их анализирует и отвечает.
  • Приложение - программа, предназначенная для выполнения определенных пользовательских задач и рассчитанная на непосредственное взаимодействие с пользователем (вспомогательная программа).
  • Операционная система - комплекс взаимосвязанных программ, предназначенных для управления ресурсами компьютера и организации взаимодействия с пользователем.
  • Оперирование - использование чего-либо при рассуждениях, подсчетах, доказательствах (пример: Джон оперировал фактами. Он оперировал информацией).
  • Специальные условные конструкции - используются для решения вопросов связанных с "Если?". Используются для принятия решений в зависимости от определенных условий.
  • Синтаксис (syntax) - это определенные команды, специальные слова и различные символы, которые используются для написания кода (правила). У каждого языка программирования свой синтаксис, то есть свои правила написания кода.
  • Синтаксическая конструкция - допустимое синтаксисом языка программирования сочетание символов, или символов и зарезервированных слов.
  • Переменная (variable) - это место хранения данных, любого типа. Можно использовать для хранения товаров, посетителей или списка пациентов в поликлинике.
  • Локальная переменная (local variable) - переменная, которая объявлена внутри { }. Такая переменная не будет существовать за пределами { } в которых она объявлена.
  • Глобальная переменная (global variable) - это переменная, которая находятся за пределами функции.
  • Объявление переменной (declaration variable) -  создание переменной в тексте программы.
  • Присваивание значения переменной - механизм связывания , позволяющий динамически изменять связи имён объектов данных (как правило, переменных) с их значениями.
  • Инициализация  переменной - это объявление переменной и присвоение ей начального значения, при помощи оператора =. Без присвоения начального значения переменная просто объявлена, а с начальным значением она еще и инициализирована (запущена в работу).
  • Интерактивность - понятие,  которое раскрывает характер и степень взаимодействия между объектами или субъектами. Элементами интерактивности являются все элементы взаимодействующей системы, при помощи которых происходит взаимодействие с другой системой или человеком (пользователем).
  • Парсинг (Parsing), синтаксический анализ (разбор) - процесс обработки и представления данных. По сути – автоматизированный сбор разрозненной информации с сайтов, ее сортировка и выдача в форме структуры (например, таблицы), при помощи парсера.
  • Парсер - специальная программа, которая выполняет парсинг (синтаксический анализ).
  • Браузерный движок (browser engine) – представляет собой программу, преобразующую содержимое веб-страницы (файлы HTML, XML, цифровое изображение и т.д.) и информацию о форматировании (в форматах CSS, XSL и т.д.) в интерактивное изображение форматированного содержимого на экране. Браузерный движок обычно используется в веб-браузерах, почтовых клиентах и других программах, нуждающихся в отображении и редактировании содержимого веб-страниц.
  • V8 – движок JS с открытым исходным кодов, распространяемый по лицензии BSD.
  • Мультимедиа (multimedia) — данные, или содержание, которые представляются одновременно в разных формах: звук, анимированная компьютерная графика, видеоряд.
  • Пустое место (White space) - переходы на новую строку и пробелы.
  • Алгоритм - система последовательных операций (в соответствии с определёнными правилами) для решения какой-нибудь задачи.
  • Рефакторинг (refactoring ) - переработка кода, равносильное преобразование алгоритмов — процесс изменения внутренней структуры программы, не затрагивающий её внешнего поведения и имеющий целью облегчить понимание её работы.
  • AJAX (Asynchronous Javascript and Xml) – технология обращения к серверу без перезагрузки страницы. За счет этого уменьшается время отклика и веб-приложение по интерактивности больше напоминает десктоп. Под AJAX подразумевают любое общение с сервером без перезагрузки страницы, организованное при помощи JS. Несмотря на то, что название технологии присутствует буква «X» (от слова XML), использовать XML вовсе не обязательно.
  • COMET – общий термин, описывающий различные техники получения данных по инициативе сервера. Можно сказать, что AJAX – это «отправил запрос – получил результат», а COMET – это «непрерывный канал, по которому приходят данные».
  • XML (eXtensible Markup Language) – расширяемый язык разметки. Язык называется расширяемым, поскольку он не фиксирует разметку, используемую в документах: разработчик волен создать разметку в соответствии с потребностями к конкретной области, будучи ограниченным лишь синтаксическими правилами языка.
  • Строка (string) - тип данных, к которому относятся любые текстовые данные.
  • Число (number) - это понятие количества, при помощи которого производятся математические действия.
  • BigInt - новый числовой примитивный тип данных в JS, позволяющий работать с числами произвольной точности. С BigInt можно безопасно хранить и обрабатывать большие целые числа даже за пределами максимального безопасного целочисленного значения Number .
  • Логический тип данных (Boolean) - примитивный тип данных в информатике, который может принимать два возможных значения, иногда называемых истиной ( true ) и ложью ( false ).
  • Null - формирует отдельный тип данных, который содержит только значение null. В JS Null не является "ссылкой на несуществующий объект" или "нулевым указателем" как в некоторых других языках. Это просто специальное значение, которое представляет собой "ничего, пусто или значение неизвестно".
  • Undefined - специальное значение, которое формирует тип из самого себя так же, как и Null. Оно означает, что "Значение не было присвоено" или "Не определено". Если переменная объявлена, но ей не присвоено никакого значения, то ее значением будет Undefined.
  • Symbol - это уникальный и неизменяемый тип данных, который может быть использован как идентификатор для свойств Object.
  • Специальные числовые значения - эти значения формально принадлежат типу «число», хотя, конечно, числами в их обычном понимании не являются - Infinity, -Infinity & NaN.
  • Infinity (бесконечность) - представляет собой математическую бесконечность. Это особое значение, которое больше любого числа. Это число можно получить в результате деления на 0.
  • NaN (not a number) (не число) - означает вычислительную ошибку. Это результат неправильной или неопределенной математической операции, которая выполняется при условии, что один из операндов не является значением с типом Number.
  • Оператор typeof - возвращает тип аргумента. Это полезно, когда нужно обработать значения различных типов данных по разному, или просто нужно сделать проверку типа данных.
  • Математическая бесконечность - общее понятие реализаций идеи бесконечности в математике, означающий "без конца".
  • Интернет (internet) - всемирная система объединенных компьютерных сетей для хранения и передачи информации.
  • Клиент (client) - аппаратный или программный компонент вычислительной системы, посылающий запросы серверу. Программа (веб-браузер, клиент E-mail, мессенджер, стриминговый сервис...), являющаяся клиентом, взаимодействует с сервером, используя определенный протокол. Она может запрашивать с сервера какие-либо данные, манипулировать данными непосредственно на сервере, запускать на сервере новые процессы и т.п.
  • Сервер (server) - программный компонент вычислительной системы, выполняющий сервисные (обслуживающие) функции по запросу клиента, предоставляя ему доступ к определенным ресурсам и услугам. Сервером можно считать выделенным компьютером, всегда подключенным к Интернету, единственной целью которого является предоставление контента.
  • Протокол передачи данных (data transfer protocol) - набор определенных правил или соглашений интерфейса логического уровня, который определяет обмен данными между различными программами. Эти правила задают единообразный способ передачи сообщений и обработки ошибок.
  • Сетевой протокол (network protocol) - набор правил и действий (очередности действий), позволяющий осуществлять соединение и обмен данными между двумя и более включенными в сеть устройствами.
  • Уникальный указатель ресурса (Uniform Resource Locator, URL) - система унифицированных адресов электронных ресурсов, или единообразный определитель местонахождения ресурса. Используется как стандарт записи ссылок на объекты в Интернете (гипертекстовые ссылки).
  • Веб-страница (web page) - документ или информационный ресурс всемирной паутины, доступ к которому осуществляется с помощью веб-браузера. Типичная веб-страница представляет собой текстовый файл в формате HTML, который может содержать ссылки на файлы в других форматах (текст, графические изображения, мультимедиа, БД и т.д.), а также гиперссылки для быстрого перехода на другие веб-страницы или доступа к ссылочным файлам.
  • Веб-сайт (website) - одна или несколько логически связанных между собой веб-страницы; также место расположения контента сервера. Обычно сайт в интернете, представляет собой массив связанных данных, имеющих уникальный адрес и воспринимаемый пользователями как единое целое.
  • Веб-браузер (web browser) - прикладное программное обеспечение для просмотра страниц, содержания веб-документов, компьютерных файлов и их каталогов; управления веб-приложениями, а также для решения других задач. В глобальной сети браузеры используют для запроса, обработки, манипулирования и отображения содержания веб-сайтов.
  • Хостинг (hosting) - услуга по предоставлению ресурсов для размещения информации на сервере, постоянно имеющем доступ к сети (обычно Интернет).
  • Система управления взаимодействием с поставщиком (supplier relationship management system, CRM) - система, помогающая контролировать все каналы коммуникаций с клиентами и автоматизировать продажи. Вместо табличек Excel, мессенджеров, множества документов и прочего.
  • Полиморфизм (polymorphism) - в языках программирования и теории типов - способность функции обрабатывать данные разных типов.
  • Инвариантность (invariant) - в программировании - логическое выражение, истинность которого сохраняется после каждого прохода тела цикла. В ООП - выражение, определяющее непротиворечивое внутреннее состояние объекта.
  • Открытое программное обеспечение (open-source software) - программное обеспечение с открытым исходным кодом. Исходный код таких программ доступен для просмотра, изучения и изменения, что позволяет убедиться в отсутствии уязвимостей и неприемлемых для пользователя функций (к примеру, скрытого слежения за пользователем программы), принять участие в доработке программы, использовать код для создания новых программ и исправления в них ошибок — через заимствование исходного кода, если это позволяет совместимость лицензий.
  • Старый код (legacy code) - устойчивое выражение, обозначающее старый код без каких-либо пояснений как в нём разобраться.
  • Инкапсуляция (encapsulation) - в информатике размещение в одном компоненте данных и методов, которые с ними работают. В реализации большинства языков программирования, обеспечивает механизм сокрытия, позволяющий разграничивать доступ к различным компонентам программы.
  • Рекурсия (recursion ) - определение, описание, изображение какого-либо объекта или процесса внутри самого этого объекта или процесса, то есть ситуация, когда объект является частью самого себя.
  • Нативное (собственное) приложение (native app) - прикладные программы, которые разрабатываются для использования на определённой платформе или на определённом устройстве. Одно из преимуществ нативных приложений — то, что они оптимизированы под конкретные операционные системы, которая задана или предоставлена, поэтому они могут работать корректно и быстро. Также они имеют доступ к аппаратной части устройств, то есть могут использовать в своих функциях камеру и микрофон смартфона, акселерометрию, геолокацию, адресную книгу, плеер и т.д.
  • Одностроничное приложение (single page application, SPA) - это веб-приложение, либо веб-сайт, использующий единственный HTML-документ как оболочку для всех веб-страниц и организующий взаимодействие с пользователем через динамически подгружаемые HTML, CSS & JavaScript, обычно посредством AJAX. Напоминает нативное приложение, с той лишь разницей, что исполняется в рамках браузера, а не в собственном процессе операционной системы.
  • Программный интерфейс приложения (application programming interface, API) - совокупность инструментов и функций в виде интерфейса для создания новых приложений, благодаря которому одна программа будет взаимодействовать с другой. API упрощает процесс программирования при создании приложений, абстрагируя базовую реализацию и предоставляя только объекты или действия, необходимые разработчику. Если графический интерфейс для почтового клиента может предоставить пользователю кнопку, которая выполнит все шаги для выборки и выделения новых писем, то API для ввода/вывода файлов может дать разработчику функцию, которая копирует файл из одного места в другое, не требуя от разработчика понимания операций файловой системы, происходящих за кулисами.
  • Шаблон проектирования, паттерн (design pattern) - в разработке программного обеспечения - повторяемая архитектурная конструкция, представляющая собой решение проблемы проектирования в рамках некоторого возникающего контекста. Обычно шаблон не является законченным образцом, который может быть прямо преобразован в код; это лишь пример решения задачи, который можно использовать в различных ситуациях.
  • Инкапсуляция (encapsulation) - в информатике, размещение в одном компоненте данных и методов, которые с ними работают. Обеспечивает механизм сокрытия, позволяющий разграничивать доступ к различным компонентам программы.
  • Сокрытие (concealment) - принципе проектирования, заключающийся в разграничении доступа различных частей программы ко внутренним компонентам друг друга.

JavaScript concepts:

  • Node или Node.js - программная платформа, основанная на движке V8 (транслирующем JS в машинный код), превращающая JS из узкоспециализированного языка в язык общего назначения. Node.js добавляет возможность JS взаимодействовать с устройствами ввода-вывода через свой API, написанный на C++, подключать другие внешние библиотеки, написанные на разных языках, обеспечивая вызовы к ним из JS-кода.

React concepts:

  • React - это JS-библиотека для разработки пользовательского интерфейса. Изначально был спроектирован так, чтобы его можно было внедрять в проект постепенно. Другими словами, можно начать с малого и использовать только тот функционал React, который необходим в данный момент.
  • JSX (JavaScript XML) - это расширение синтаксиса JavaScript, которое позволяет использовать HTML-подобный синтаксис для описания структуры интерфейса.
  • React-компонент () - это функция, которая возвращает разметку JSX. Компоненты помогают разбить интерфейс на независимые части, про которые легко думать в отдельности. Их можно складывать вместе и использовать несколько раз.
  • Пропсы (props) - это входные данные React-компонентов, передаваемые от родительского компонента к дочернему.
  • Хуки (Hooks) - это функции, с помощью которых можно "подцепиться" к состоянию и методам жизненного цикла React из функциональных компонентов. Хуки не работают внутри классов - они дают возможность использования React без классов.
